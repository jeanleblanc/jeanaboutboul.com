## Categories

A **category** $\mathcal{C}$ consists of:

- A collection of **objects** $\text{Ob}(\mathcal{C})$
- For each pair of objects $A, B$, a set of **morphisms** $\text{Hom}(A, B)$
- Composition of morphisms: if $f: A \to B$ and $g: B \to C$, then $g \circ f: A \to C$
- An identity morphism $\text{id}_A: A \to A$ for each object

subject to associativity and identity laws.

## Functors

A **functor** $F: \mathcal{C} \to \mathcal{D}$ maps objects and morphisms
between categories while preserving structure:

$$
F(g \circ f) = F(g) \circ F(f), \qquad F(\text{id}_A) = \text{id}_{F(A)}
$$

### Example: The List Functor

In Haskell, `[]` is a functor from **Hask** to **Hask**:

```haskell
fmap :: (a -> b) -> [a] -> [b]
fmap f []     = []
fmap f (x:xs) = f x : fmap f xs
```

## Natural Transformations

A **natural transformation** $\eta: F \Rightarrow G$ between functors
$F, G: \mathcal{C} \to \mathcal{D}$ assigns to each object $A$ a morphism
$\eta_A: F(A) \to G(A)$ such that the following diagram commutes for every
morphism $f: A \to B$:

$$
G(f) \circ \eta_A = \eta_B \circ F(f)
$$

### Example: `head` as a Natural Transformation

The function `head :: [a] -> Maybe a` is a natural transformation from the
List functor to the Maybe functor.

## Why This Matters

Category theory provides a unified language for describing mathematical
structures and their relationships — and has deep connections to logic
(via the Curry-Howard-Lambek correspondence) and computation.
