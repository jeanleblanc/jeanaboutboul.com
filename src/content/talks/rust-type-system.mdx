## Why Rust?

Rust provides memory safety guarantees at compile time through its ownership
system — without the runtime overhead of a garbage collector.

## The Three Rules

### Ownership

Every value in Rust has exactly one owner. When the owner goes out of scope,
the value is dropped:

```rust
fn main() {
    let s = String::from("hello");  // s owns the String
    let t = s;                       // ownership moves to t
    // println!("{}", s);            // ERROR: s no longer valid
    println!("{}", t);               // OK
}
```

### Borrowing

References allow you to access data without taking ownership:

```rust
fn length(s: &String) -> usize {
    s.len()  // s is borrowed, not consumed
}
```

### Lifetimes

Lifetimes ensure that references never outlive the data they point to:

```rust
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

## Algebraic Data Types

Rust's `enum` types are full algebraic data types (sum types), enabling
pattern matching:

```rust
enum Shape {
    Circle(f64),
    Rectangle(f64, f64),
}

fn area(s: &Shape) -> f64 {
    match s {
        Shape::Circle(r) => std::f64::consts::PI * r * r,
        Shape::Rectangle(w, h) => w * h,
    }
}
```

## Zero-Cost Abstractions

Rust's trait system provides polymorphism that compiles down to static dispatch
with no runtime overhead — the same performance as hand-written C code.
